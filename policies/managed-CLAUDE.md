# Organization Standards

These rules apply to all Claude Code users in the organization.

## Security

- Never commit secrets, API keys, tokens, or credentials to version control
- Never run destructive commands (rm -rf, DROP TABLE, etc.) without explicit user confirmation
- Never access files outside the current project directory

## Code Quality

- Always run the project's test suite before marking work complete
- Follow the project's existing code style and formatting conventions
- Write tests for all new functionality

## Compliance

- Do not install packages from untrusted sources
- Do not send code or data to external services without user approval
- Log all significant actions for audit purposes
