# Communication Plan

## Pre-Launch (2 weeks before Cohort 1)

### Audience: Engineering Leadership

- **Format**: 30-minute briefing
- **Content**: Business case, cost projections, security controls, timeline
- **Owner**: Platform engineering lead
- **Approval required**: Yes

### Audience: Security Team

- **Format**: Technical review meeting
- **Content**: Architecture diagram, data flow, managed settings, audit capabilities
- **Owner**: Platform engineering + security lead
- **Approval required**: Yes

## Cohort 1 Launch

### Audience: Cohort 1 Developers

**Email:**

```
Subject: You're invited to the Claude Code pilot program

Hi [name],

You've been selected for the Claude Code pilot (Cohort 1, starting [date]).

What is Claude Code? An AI coding assistant that runs in your terminal,
understands your codebase, and helps with development tasks.

What to expect:
- Access will be provisioned on [date]
- Training session: [date/time/link]
- Support channel: #claude-code-cohort-1
- Feedback survey every 2 weeks

Getting started:
1. Claude Code will be available after [date]
2. Attend the training session or review the self-paced materials at [link]
3. Start with small tasks (bug fixes, test writing) before larger features

Questions? Reply to this email or post in #claude-code-cohort-1.
```

### Audience: Cohort 1 Managers

**Email:**

```
Subject: Claude Code pilot -- your team members are participating

Hi [manager name],

[Developer names] from your team have been selected for the Claude Code
pilot program starting [date].

What this means for your team:
- Developers will spend ~30 minutes on initial training
- Expect a learning curve in the first week
- Monitor for any changes in PR quality or velocity

How you can help:
- Encourage experimentation during the pilot
- Share feedback with us (positive or negative)
- Flag any concerns to #claude-code-platform

We'll share metrics with you bi-weekly.
```
